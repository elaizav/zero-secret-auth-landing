Повний опис дослідження

Сторінка містить розширений академічний опис теми, методології, завдань і очікуваних результатів бакалаврської роботи.

Тема дослідження: «Методологія та інструментальні засоби розробки систем автентифікації за моделлю Zero Secret Server Authentication» / “Methodology and Instrumental Tools for Developing Authentication Systems Based on the Zero Secret Server Authentication Model”.

01 · Зміст

Предмет, мета і задачі дослідження

Короткий опис

Дослідження присвячене проєктуванню систем автентифікації, у яких сервер мінімізує або усуває необхідність зберігання секретів користувача. Центральна ідея полягає в перенесенні частини критично важливих механізмів на клієнтську сторону та в побудові протоколу, де сервер працює з верифікаційними артефактами, сесіями та службовими метаданими.

Мета

Сформувати методологію аналізу та добору інструментальних засобів для розробки безпечної системи автентифікації за моделлю Zero Secret Server Authentication, а також підготувати описову основу для майбутнього демонстраційного прототипу.

Основні завдання

  1. Проаналізувати сучасні схеми автентифікації та їхні типові ризики.
  2. Описати принципи Zero Secret Server Authentication і межі застосування моделі.
  3. Порівняти інструментальні підходи для вебреалізації з погляду безпеки й практичності.
  4. Сформувати цільову архітектуру системи та описати її ключові ролі.
  5. Підготувати структуру публічних матеріалів і базу для наступної реалізації прототипу.

Очікувані результати

  • Узгоджений опис архітектури без серверного зберігання секретів користувача.
  • Перелік інструментів і практик для безпечної вебреалізації.
  • Обґрунтування вибору підходів для клієнтської криптографії та сесійної логіки.
  • Публічний академічний лендінг і сторінка матеріалів, придатні для GitHub Pages.

02 · Архітектурна ідея

Як описується zero-secret підхід у межах цього проєкту

Пояснення схеми

Узагальнений сценарій передбачає, що користувач взаємодіє з клієнтським застосунком, який виконує обчислення, пов'язані з локальним секретом. Сервер отримує лише ті дані, що достатні для перевірки запиту або видачі сесійного результату, але не вимагають збереження початкового секрету в явному вигляді.

Матеріал окреслює дослідницький напрям, архітектурну логіку та критерії для подальшої реалізації демонстраційного прототипу.

03 · Межі публікації

Що ця сторінка стверджує, а чого не стверджує

Що містять матеріали

  • Тема бакалаврського дослідження в українській та англійській формах.
  • Короткий опис, мета, завдання, методологія та очікувані результати.
  • Візуальні матеріали для академічної презентації концепції.
  • Публічний репозиторій зі статичним кодом лендінгу та git-історією.

Межі подання

  • Сторінка подає структуру й зміст дослідження без надмірних тверджень щодо стадії реалізації.
  • Опис зосереджений на темі, методології, архітектурній ідеї та очікуваних результатах.
  • Контактний блок містить лише публічно доступні посилання автора.
  • Подані матеріали призначені для академічної презентації на GitHub Pages.
