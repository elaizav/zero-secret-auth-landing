Methodology and Instrumental Tools for Developing Authentication Systems Based on the Zero Secret Server Authentication Model

The project studies how authentication systems can be designed so that the server does not retain user secrets, while critical security logic is supported by client-side cryptographic mechanisms and a controlled verification flow.

01 · Relevance

Why the topic matters

Limits of traditional schemes

Many classic authentication systems rely on server-side storage of passwords, derived secrets or related sensitive artifacts. This enlarges the attack surface and makes database leaks more damaging.

Research focus

The Zero Secret Server Authentication model is studied as a way to reduce trust in server-side secret retention and strengthen the role of client-side cryptography, bounded protocol steps and minimal exposure of sensitive material on the backend.

02 · Goal and tasks

Research goal, tasks and expected outcomes

Research goal

To investigate methodological approaches and development tools for authentication systems based on the Zero Secret Server Authentication model and to prepare the basis for a future demonstration prototype.

  1. 01

    Review modern authentication approaches and their risks related to storing user secrets.

  2. 02

    Clarify the principles of the Zero Secret Server Authentication model and its practical scope.

  3. 03

    Select development tools suitable for a secure web implementation with client-side cryptography.

  4. 04

    Define the target architecture and describe the roles of the client, server and verification artifacts.

  5. 05

    Prepare a public materials structure and a foundation for a future demonstration prototype.

Expected results

An aligned design methodology, a grounded tooling set, an architecture description without server-side secret storage and a prepared basis for a prototype concept.
